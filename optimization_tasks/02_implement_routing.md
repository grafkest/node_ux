# Задача 2: Внедрение клиентского роутинга (React Router)

## Проблема
Навигация в приложении реализована через условный рендеринг (`viewMode` state) внутри `App.tsx`.
*   Нет возможности поделиться ссылкой на конкретный экран (например, на статистику).
*   Кнопки "Назад" в браузере не работают как ожидается.
*   Все компоненты загружаются сразу (даже если используется `lazy`, логика их отображения смешана).

## Цель
Заменить state-based навигацию на полноценный роутинг с использованием `react-router-dom`.

## План действий

1.  **Установка зависимостей**
    *   `npm install react-router-dom`

2.  **Настройка роутера**
    *   Создать конфигурацию маршрутов в `src/router.tsx` (или `src/AppRoutes.tsx`).
    *   Определить маршруты:
        *   `/` -> Redirect to `/graph`
        *   `/graph` -> `GraphPage`
        *   `/stats` -> `StatsPage`
        *   `/experts` -> `ExpertsPage`
        *   `/initiatives` -> `InitiativesPage`
        *   `/tasks` -> `EmployeeTasksPage`
        *   `/admin` -> `AdminPage`
        *   `/login` -> `LoginPage`

3.  **Создание компонентов-страниц**
    *   Создать папку `src/pages`.
    *   Создать обертки для основных разделов:
        *   `src/pages/GraphPage.tsx` (содержит `GraphView` и сайдбары)
        *   `src/pages/AdminPage.tsx` (содержит `AdminPanel`)
        *   и т.д.

4.  **Обновление `LayoutShell`**
    *   Вместо пропса `currentView` использовать `useLocation` и `NavLink` для меню.
    *   Заменить условный рендеринг `{viewMode === 'graph' && ...}` на `<Outlet />`.

5.  **Рефакторинг `App.tsx`**
    *   Заменить ручной рендеринг на `<RouterProvider router={router} />`.

## Ожидаемый результат
*   URL в браузере будет соответствовать открытому экрану.
*   Будет работать история браузера (назад/вперед).
*   Улучшится разделение кода (Code Splitting) для каждого маршрута.
