# Оптимизация адаптивной верстки для "квадратных" дисплеев

## Проблема
На дисплеях с соотношением сторон, близким к квадратному (например, 5:4, 1280x1024), или просто на нешироких экранах (1200px - 1400px), текущая 3-колоночная верстка работает некорректно.
Центральная область с графом становится слишком узкой (~500-600px), что затрудняет работу с визуализацией.

## Анализ текущего состояния
*   **Breakpoint 1200px:** Ниже этой ширины включается мобильный вид.
*   **Breakpoint 1440px:** Ниже этой ширины используются ширины колонок 280px и 320px.
*   **Проблема:** В диапазоне 1200px - 1366px боковые панели и отступы занимают слишком много места (~650px из доступных 1280px), оставляя графу менее 50%.

## Список действий для исправления (Фокус на масштабировании и отступах)

### 1. Компактный режим для диапазона 1200px - 1440px
Необходимо уменьшить "вес" интерфейса за счет сужения панелей и уменьшения отступов, сохраняя все элементы видимыми (без скрытия правой панели).

*   **Действие:** В `App.module.css` для медиа-запроса `@media (max-width: 1440px)` применить более агрессивное уменьшение размеров:
    *   **Sidebar (Левая панель):** Уменьшить с 280px до **220px-240px**.
    *   **Details (Правая панель):** Уменьшить с 320px до **260px-280px**.
    *   **Grid Gap:** Уменьшить с 24px до **12px**.
    *   **Padding (Внешние отступы):** Уменьшить с 24px до **12px**.

*   **Результат:**
    *   Было: 24px (pad) + 280px (side) + 24px (gap) + Graph + 24px (gap) + 320px (details) + 24px (pad) = ~700px обвязки.
    *   Станет: 12px + 240px + 12px + Graph + 12px + 280px + 12px = ~568px обвязки.
    *   Выигрыш: **~130px** ширины для графа.

### 2. Уменьшение масштаба интерфейса (Zoom/Scaling)
Для сохранения информационной емкости при уменьшении ширины панелей можно немного уменьшить размер шрифта и элементов.

*   **Действие:** Для разрешения 1200px-1366px уменьшить базовые размеры:
    *   Уменьшить внутренние отступы в карточках (`padding` внутри `sidebar` и `details`) с 16px/24px до **12px/16px**.
    *   При необходимости уменьшить размер шрифта на 1px для второстепенных элементов.

### 3. Адаптация GraphView
*   **Действие:** Убедиться, что `GraphView` корректно ресайзится и центрирует граф при изменении ширины контейнера (проверить работу `ResizeObserver`).

## Рекомендуемый план реализации
1.  Настроить CSS Grid в `App.module.css` для диапазона 1200px-1440px с новыми компактными размерами (отступы 12px, колонки 240px/280px).
2.  Уменьшить внутренние отступы (`padding`) в компонентах боковых панелей для этого разрешения, чтобы контент не выглядел зажатым.
